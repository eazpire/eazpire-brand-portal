Brand API Documentation

Machine access to your brand on brand.eazpire.com — catalog, dual-publish, team, webhooks, and profile.

Authentication

Create an eazpire API key in the Brand Portal under Settings → eazpire API keys. The full key is shown once.

Send the key on every request:

  • Authorization: Bearer eaz_brand_…
  • or X-Eazpire-Brand-Key: eaz_brand_…

Portal browser sessions use a cookie instead and have full access (*). Keys are scoped to one brand and never store the raw secret after creation.

Not the same as: BYO Printify/Shopify connections, or “Link eazpire Account” (Customer Account OAuth for Creator design).

Scopes

When creating a key you can pick scopes, or leave defaults (all listed scopes). Use * for full access.

Endpoints

Base URL: https://brand.eazpire.com. Prefer versioned paths; ?op= aliases still work.

POST /api/v1/brand brand:write Body: { name?, handle?, tagline?, about? }

Webhooks

Orders

Orders for your products listed on eazpire (dual-published). Not orders from your own BYO Shopify shop. Read-only — no fulfill / refund. Poll this API; inbound Shopify→brand order webhooks are not offered in MVP.

Method / pathScopeNotes
GET /api/v1/orders orders:read Optional ?limit (1–50, default 25), ?since_id, ?created_at_min (ISO), ?financial_status, ?fulfillment_status, ?status (any|open|closed|cancelled). Returns only orders that include your dual-published line items.
GET /api/v1/orders/{id} orders:read Detail + brand line items + customer/shipping. 404 if the order has no items for this brand.

/api/v1/memberships and API key CRUD require a portal session (not API keys). Memberships need Link eazpire Account for Creator design. Suspended brands receive brand_suspended on orders.

Webhooks

Register an HTTPS endpoint (localhost HTTP allowed for local testing). eazpire POSTs JSON when catalog events fire. Delivery retries up to 3 times on timeouts / 5xx; after repeated failures the webhook is auto-disabled.

Events

EventWhen
product.publishedAfter successful dual-publish to eazpire
product.unpublishedAfter unpublish / draft on eazpire
product.updatedAfter local product title/status write
product.syncedAfter Printify sync completes (includes synced count)
webhook.pingManual test from portal or /test

Headers

  • X-Eazpire-Event — event name
  • X-Eazpire-Delivery-Id — unique delivery id
  • X-Eazpire-Signature — sha256=<hex> HMAC-SHA256 of the raw request body using your signing secret
  • Content-Type: application/json

Payload example

{
  "id": "del_…",
  "event": "product.published",
  "created_at": 1710000000000,
  "brand_id": "b_…",
  "data": {
    "product_id": "bp_…",
    "printify_product_id": "…",
    "title": "…",
    "eazpire_shopify_product_id": "…",
    "eazpire_handle": "…",
    "dual_publish_status": "published"
  }
}

Verify signature (Node.js)

import crypto from "crypto";

function verifyEazpireSignature(rawBody, signatureHeader, secret) {
  const expected = "sha256=" + crypto
    .createHmac("sha256", secret)
    .update(rawBody, "utf8")
    .digest("hex");
  const a = Buffer.from(String(signatureHeader || ""));
  const b = Buffer.from(expected);
  return a.length === b.length && crypto.timingSafeEqual(a, b);
}

HTTPS URLs only (except http://localhost / 127.0.0.1). Private / metadata IPs are blocked. Never log the full signing secret.

Examples

List products

curl -sS "https://brand.eazpire.com/api/v1/products" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY"

Update brand tagline

curl -sS -X POST "https://brand.eazpire.com/api/v1/brand" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"tagline":"Made for creators"}'

Invite a team member

curl -sS -X POST "https://brand.eazpire.com/api/v1/team/invite" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"email":"creator@example.com","publish_mode":"review"}'

Publish to eazpire

curl -sS -X POST "https://brand.eazpire.com/api/v1/products/publish" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"product_ids":["bp_…"]}'

Register a webhook

curl -sS -X POST "https://brand.eazpire.com/api/v1/webhooks" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/hooks/eazpire","events":["product.published","product.unpublished"]}'

List eazpire orders (dual-published products)

curl -sS "https://brand.eazpire.com/api/v1/orders?limit=25" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY"

Get one order

curl -sS "https://brand.eazpire.com/api/v1/orders/ORDER_ID" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY"
