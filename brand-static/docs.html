Authentication

Create an eazpire API key in the Brand Portal under Settings → eazpire API keys. The full key is shown once.

Send the key on every request:

  • Authorization: Bearer eaz_brand_…
  • or X-Eazpire-Brand-Key: eaz_brand_…

Portal browser sessions use a cookie instead and have full access (*). Keys are scoped to one brand and never store the raw secret after creation.

Not the same as: BYO Printify/Shopify connections, or “Link eazpire Account” (Customer Account OAuth for Creator design).

Scopes

When creating a key you can pick scopes, or leave defaults (all listed scopes). Use * for full access.

ScopeAllows
overview:readBrand overview + stats
brand:readRead brand profile
brand:writeUpdate name, tagline, about, handle (no delete)
connections:readConnection status (no secrets)
products:readList / get products
products:writeUpdate local product title/status
products:syncSync catalog from BYO Printify
products:publishPublish / unpublish on eazpire
team:readList team members
team:inviteInvite creators by email
team:writeUpdate publish_mode / revoke members
*All scopes

Endpoints

Base URL: https://brand.eazpire.com. Prefer versioned paths; ?op= aliases still work.

Overview & brand

Method / pathScopeNotes
GET /api/v1/overview overview:read Profile + product/member counts + connection summary
GET /api/v1/brand brand:read Brand profile only
POST /api/v1/brand brand:write Body: { name?, handle?, tagline?, about? }

Connections

Method / pathScopeNotes
GET /api/v1/connections connections:read Status, shop ids/names — never API secrets

Products

Method / pathScopeNotes
GET /api/v1/products products:read Optional ?status=active|draft
GET /api/v1/products/{id} products:read Single catalog row
POST /api/v1/products/{id} products:write Body: { title?, status? } — local metadata only; Printify sync may overwrite
POST /api/v1/products/sync products:sync Refresh from BYO Printify
POST /api/v1/products/publish products:publish { product_id } or { product_ids } or { limit }
POST /api/v1/products/unpublish products:publish { product_ids } or { all: true }

Team

Method / pathScopeNotes
GET /api/v1/team team:read Members (excludes owner row)
POST /api/v1/team/invite team:invite { email, publish_mode?: "review"|"auto_publish" }
POST /api/v1/team/update team:write { member_id, publish_mode?, status? }
POST /api/v1/team/revoke team:write { member_id }

Examples

List products

curl -sS "https://brand.eazpire.com/api/v1/products" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY"

Update brand tagline

curl -sS -X POST "https://brand.eazpire.com/api/v1/brand" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"tagline":"Made for creators"}'

Invite a team member

curl -sS -X POST "https://brand.eazpire.com/api/v1/team/invite" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"email":"creator@example.com","publish_mode":"review"}'

Publish to eazpire

curl -sS -X POST "https://brand.eazpire.com/api/v1/products/publish" \
  -H "Authorization: Bearer eaz_brand_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"product_ids":["bp_…"]}'
